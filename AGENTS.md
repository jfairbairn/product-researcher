# BDD Project Configuration

## BDD Method

This project uses outside-in BDD with strict red-green-refactor enforcement.

Rules (enforced by the bdd-enforcer extension):
1. Never write to production paths before a failing test is confirmed
2. Never write production code before running tests and seeing them fail
3. Write the minimum code to make the test pass — no speculative implementation
4. Refactor only when tests are green
5. Every phase boundary (RED, GREEN, REFACTOR complete) is a git commit

## Stack

Language: TypeScript (ESM, NodeNext module resolution)
Test framework: Vitest
Test command: npm test
Acceptance layer: plain Vitest describe (integration tests against local server)
HTTP testing: native fetch / supertest
UI testing: @testing-library/svelte (when needed)
DB fixtures: Vitest beforeEach with Drizzle test transactions

## Architecture

Monorepo (npm workspaces):
  frontend/   SvelteKit + @sveltejs/adapter-node
  backend/    Fastify API server + BullMQ research worker

Infrastructure:
  PostgreSQL 16   — persistent knowledge graph (seeds, nodes, edges, runs)
  Redis 7         — BullMQ job queue for continuous research runs

LLM providers (switchable per agent run):
  Anthropic SDK   — Claude models
  OpenAI SDK      — GPT / o-series models

## File Layout

Production code:  backend/src/    frontend/src/
Test/spec code:   backend/tests/  frontend/src/**/*.test.ts
Fixtures:         backend/tests/fixtures/
DB schema:        backend/src/db/schema.ts
Migrations:       backend/drizzle/    (generated by drizzle-kit, committed to git)

## BDD Config (.pi/bdd.config.json)

```json
{
  "productionPaths": ["backend/src/", "frontend/src/"],
  "testPaths": ["backend/tests/", "frontend/src/"],
  "testFilePatterns": ["\\.test\\.", "\\.spec\\."],
  "testCommand": "npm test"
}
```

## Naming Conventions

Spec files mirror source files:
  backend/src/api/seeds.ts          → backend/tests/api/seeds.test.ts
  backend/src/agents/researcher.ts  → backend/tests/agents/researcher.test.ts
  frontend/src/routes/+page.svelte  → frontend/src/routes/+page.test.ts

## Test Data

Use builder functions for test data:
  backend/tests/builders/seedBuilder.ts

DB state: reset per test using Drizzle transactions (rollback after each test).

## Documentation

Minimum-viable documentation per component: README.md in component directory.
Six categories: purpose, what it does, how it does it, status, decisions, roadmap.

Project roadmap: ROADMAP.md at root.
Product success conditions: PRODUCT.md at root.

## Release Gate

Release config: .pi/release.config.json
Staging URL: https://staging.YOURDOMAIN (Hetzner + Coolify — not yet provisioned)
Production URL: https://YOURDOMAIN
Coolify API token: COOLIFY_API_TOKEN (environment variable)

Commands: /release — show gate status
Tools: check_release_readiness, mark_gate_passed

Six gates between IDLE and deployed:
  1. Functional correctness (auto — BDD cycle)
  2. Security (auto — security_scan)
  3. Non-functional (skipped — no load tests yet)
  4. Pre-production staging (auto — deploy + migrate + health + test)
  5. Measurement readiness (auto — check_success_conditions)
  6. Rollback readiness (human checklist)

## Local Development

Start infrastructure:  docker compose up -d
Install deps:          npm install
Copy env:              cp .env.example .env  (fill in API keys)
Run API server:        npm run dev -w @product-researcher/backend
Run research worker:   npm run dev:worker -w @product-researcher/backend
Run frontend:          npm run dev -w @product-researcher/frontend
Run all:               npm run dev

DB migrations (dev):   npm run db:generate -w @product-researcher/backend
                       npm run db:migrate -w @product-researcher/backend
